# Y-Agent 项目愿景

## 项目定位

Y-Agent 不是一个万能的 AGI，也不是一个通用聊天机器人。它是一个**为未来 LLM 设计的 Agent 框架**，致力于通过精心的底层设计，构建一个不会被时代淘汰的、可持续演进的智能体系统。

## 设计哲学

### 长期主义

我们相信：**LLM 会持续迭代，但 Agent 架构不应该持续重构**。

正如工作流引擎在 LLM 出现之前就已经成熟并持续发挥价值，优秀的 Agent 架构应该是稳定的、可扩展的，能够适配不同能力级别的 LLM，而不是依赖于某个特定模型的能力。

### 工程质量优先

当前市场上的许多 Agent 产品存在以下问题：

- **设计水平初级**：很多开发者使用 AI 辅助编写代码，但对底层逻辑理解不深
- **过度依赖顶尖模型**：只在 Claude 或 GPT 等顶尖模型上测试通过就发布，换成 DeepSeek 或 Qwen 后准确率明显下降
- **性能未优化**：能跑通就接受，没有追求最优的系统设计
- **Skills 臃肿且反人类**：不少 Agent 的 Skills 体量庞大，又严重依赖 LLM 能力，难以理解和维护
- **代码即工具成瘾**：偏好「代码即工具」——许多简单任务也要先生成占大量 token 的脚本来执行，既不省 token 也不省心，且执行后不积累经验，违背人类习惯，无法形成可复用的能力

Y-Agent 追求的是：

- **深度理解底层机制**：精心设计的提示词策略、工具调用协议、状态管理
- **模型无关性**：架构设计应该对不同能力的 LLM 都能提供合理的表现
- **性能与准确性并重**：在保证功能的同时，追求最优的执行效率

## 目标用户

Y-Agent 的目标用户是**懂技术的开发者**，而非需要一键解决所有问题的小白用户。

我们假设用户：
- 理解编程和系统设计的基本概念
- 能够根据自己的需求定制和扩展系统
- 愿意深入理解 Agent 的工作机制

## 核心能力

### 1. 快速场景适配

通过配置式的设计，Y-Agent 可以快速应用到各种场景：

- **配置子 Agent**：定义不同职责的子 Agent，如分析、执行、验证等
- **编排任务流**：定义任务的执行顺序、条件分支、循环逻辑
- **配置工具/Skills**：为 Agent 提供领域特定的工具集

### 2. 自进化能力

Y-Agent 不仅是一个静态的工具，它可以**让 Agent 自己进化**成特定领域的专家系统：

- **自动构建工作流**：Agent 理解需求后，自动创建、组合工作流
- **动态生成工具**：根据任务需求，Agent 可以创建新的工具或 Skills
- **知识积累**：通过 RAG、Harness 等机制，积累领域知识

### 3. 工作流混合编排

Y-Agent 支持多层次的工作流组合：

- 基础工作流可以作为更复杂工作流的一部分
- 工作流之间可以共享状态和上下文
- 支持并行执行、条件执行、循环执行等复杂编排

## 应用场景示例：同人小说创作系统

假设我们要构建一个同人小说创作助手，Y-Agent 的工作方式如下：

### 初始需求
"我想让它支持同人小说续写"

### Agent 的自动演进过程

**阶段 1：理解与分析**
- Agent 分析需求，识别出需要处理原作内容、理解角色设定、生成新内容等任务

**阶段 2：自动构建工作流 A - 原作分析**
- 自动提取小说章节结构
- 构建 RAG 系统索引原作内容
- 提取角色元数据（性格、关系、经历）
- 生成 Harness（角色行为模式、语言风格）

**阶段 3：自动构建工作流 B - 同人文生成**
- 根据用户输入的剧情方向
- 基于角色设定和 Harness
- 生成符合原作风格的同人内容

**阶段 4：扩展需求处理**
用户："我还需要输出角色设定图"

Agent 自动创建**工作流 C - 角色视觉化**：
- 读取角色元数据
- 生成适合绘图模型的 Prompt
- 调用图像生成工具
- 输出角色设定图

**阶段 5：工作流混合编排**
- 工作流 A（原作分析）→ 为 B 和 C 提供基础数据
- 工作流 B（同人文生成）和 工作流 C（角色图生成）可以并行执行
- 用户可以组合这些工作流，例如："先生成剧情，再根据剧情中的新角色生成设定图"

## 技术愿景

Y-Agent 的技术追求：

1. **架构稳定性**：核心架构设计应该能够支撑未来 5-10 年的 LLM 发展
2. **模型适配性**：从 GPT-3.5 级别到 GPT-4o/Claude Opus 级别都应该能合理工作
3. **可扩展性**：新场景、新工具、新能力应该通过配置和扩展实现，而非修改核心代码
4. **工程化**：完善的日志、监控、调试工具，让开发者能够理解和优化 Agent 的行为
5. **性能优化**：合理的并发、缓存、状态管理，追求最优的执行效率

## 价值主张

Y-Agent 的价值不在于"能做什么"，而在于：

- **如何做**：精心设计的架构和机制
- **如何扩展**：灵活的配置和编排能力
- **如何适应未来**：模型无关的设计理念
- **如何赋能开发者**：让懂技术的人快速构建自己的 AI 系统

我们不追求一个能回答所有问题的超级助手，而是追求一个能够让开发者快速构建**特定领域专家系统**的基础设施。
